The Java Asynchronous Client Platform (JACP)

by Andy Moncsek

http://code.google.com/p/jacp/

Overview

  • first ideas emerged in 2005

  • since 2010 officially an open source project under Apache License 2.0

  • idea was to create a simple to use counterpart to Eclipse RCP

  • since than the ideas and goals of the project were extended, it became my personal research project ;-)

Why create a counterpart to Eclipse RCP?

Although Eclipse RCP is a great, powerful and established platform the complexity can get a substantial risk

  • back in 2005 we had at TU Dresden a project to bring Eclipse RCP to web, when Eclipse RAP was released the project was canceled

  • a lot of student workers participated in this project and the quality got more and more worse

  • major problem: basic concepts are easy to understand but beyond workspace, perspective, view and editor students created "self made" solutions

  • currently Eclipse Foundation is working on e4 project to address most of those complexity issues

What are the goals of JACP?

  1. create a simple API that match following needs:

    • it defines a minimal set of abstract components to create workbench, perspectives and components
    • it defines communication between components
    • it is basic enough to be implemented in different Toolkits like Swing, SWT, Vaadin or JavaFX 2.0

  2. create a „Actor Like“ approach to run/handle components

    • every component is handled on message call
    • components are handled in separate threads
    • components can be state- less/ful

  3. create Peer-To-Peer based approach to handle component messaging on other/remote instances

    • every component is handled on message call
    • components are handled in separate threads
